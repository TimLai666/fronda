## What is done and committed

### `editor-shell-parity` (commit `b5e25d1a`)
Reworked `crates/app_shell_gpui` so the editor matches Swift `EditorView.swift`
structurally. Verified: `cargo test --workspace` green, desktop bin builds,
Windows screenshot confirms the new layout + menu bar + rail icons.

- `pane_tree.rs` (new, pure + unit-tested): builds the preset layout tree.
  - `default`: Agent | (Media | Preview | Inspector upper 70% / **Toolbar+Timeline
    full-width lower 30%**).
  - `media`: Agent | Media(30%) | (Preview | Inspector 55% / Toolbar+Timeline).
  - `vertical`: Agent | (Media | Inspector 55% / Toolbar+Timeline)(left 50%) | Preview.
  - Agent is always the outermost left column. `pane::apply_preset` no longer
    mutates visibility (the old `media` preset that hid inspector/timeline/agent
    was an invention — removed).
- `pane_resize.rs` (new, pure + unit-tested): `clamp_resize` for the four
  draggable dividers (agent 240–640, media ≥ 280 + rail, inspector ≥ 150,
  timeline 100–700, Preview never squeezed below 400 wide / 320 tall).
  `initial_timeline_height` = 30% (`default`) / 45% (`media`,`vertical`).
- `editor_view.rs`: `pane_card` shell (surface rounded card + `PANEL_GAP/2`
  inset on base → visible 5px seam) + tree-driven render + divider anchors.
- `app_root.rs`: `PaneResizeDrag` machinery (generalized from the old
  timeline-only drag), stores agent/media/inspector width + timeline height,
  clamps on drag; welcome overlay hoisted to window level, centered + clamped;
  window origin clamped onto the display.
- `preview_view.rs`: panel bg → SURFACE + centered empty-state canvas rect.
- `media_panel_view.rs` + `assets/icons/{captions,music_note}.svg`: rail tabs
  render SVG icons (folder/captions/music) instead of "M"/"C"/"♪".
- `menu.rs`: single-source menu labels + `menu_keybinding_shortcuts()`
  (Ctrl-modifier chords, excluding text-input-owned Ctrl+A/C/V/X/Z/Y) +
  `display_hint`. `global_shortcuts.rs`: `RunMenuAction` + `bind_menu_shortcut_keys`
  (non-macOS only). `titlebar_view.rs`: File/Edit/View/Window/Help menu bar
  (non-macOS) emitting `TitleBarEvent::RunMenu`; macOS keeps the system menu.
- Specs: `openspec/changes/editor-shell-parity/` (proposal/design/spec/tasks,

## Known blocker: gpui-ce Windows platform layer (out of scope, next change)

Diagnosed in `gpui_windows` (the vendored gpui-ce git checkout, not our code):
- `events.rs` `handle_mouse_down_msg`: the input callback is `take()`n during
  dispatch; events arriving mid-dispatch are dropped (`return Some(1)`). This is
  the most likely cause of intermittent click loss.
- `handle_calc_client_size`: maximized `NCCALCSIZE` adds `frame_thickness` to
  `top` — a plausible hit-test offset when maximized.
- `handle_size_change`: a failed `renderer.resize` sets `invalidate_devices` →
  black screen (seen after external `ShowWindow` restore).

**Measured on Windows (before the fix, original diagnosis):** programmatic
"New Project" clicks succeeded ~1 in 5 attempts; the one success crashed
silently ~10s after entering the editor. Divider drags, by contrast, fired
their move events reliably in the post-fix diagnostic run. Fixing this needs a
gpui-ce fork or vendor patch — track it as its own change. On macOS this path
is different code; the app should be drivable normally.

## How to continue on macOS

```bash
# Rust app (primary):
cargo run -p fronda-app-shell-gpui --features desktop-app --bin fronda
cargo test --workspace                       # must stay green
cargo check -p fronda-app-shell-gpui --features desktop-app --bin fronda

# Swift baseline to compare against, side by side:
swift build
swift run
```

Dev seams added for verification (env vars, read in `app_root.rs`):
- `FRONDA_OPEN_EDITOR=1` — boot straight into an empty editor (skips Home; same
  path as the New Project button). Useful to bypass the Windows click-loss when
  screenshotting; harmless on macOS.
- `FRONDA_WINDOW_SIZE=WxH` — override the initial window size.

These two seams are dev-only conveniences; consider removing them (or gating
behind a debug cfg) before any release build.
