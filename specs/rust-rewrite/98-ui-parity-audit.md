# 98 — UI parity audit (Swift ↔ Rust), 2026-07-05

Full-surface structural/interaction audit of every Swift view vs its gpui
counterpart (agent sweep, verified file-by-file). Drives the /goal "UI 要跟
Swift 版完全一模一樣、真的可以使用". Pixel-level (spacing/color/type) diffs
are NOT covered here — structural/interaction parity first.

Systemic headline: the AI generation/edit surface (rows 1,3,4,5,11) renders a
plausible shell but was functionally hollow — rows 1/2/3/4/5/6 became
functional in 2026-07-10 changes (backend/host seams still gated); row 7
context menus landed for project cards AND media asset/folder tiles. The
cross-view seams left open by the parallel streams (rows 3/4 chat handoff,
row 8 crop→preview relay, row 13 inspector selection inputs) are now wired
in app_root (observers + handoff mailbox), 2026-07-10.
